Päivälliset — kahden viikon koonti

Nopea katselunäkymä. Muokkaa reseptejä Viikko 1 / Viikko 2 -välilehdillä.

Päivitetty: —

GitHub Personal Access Token

Token tallennetaan vain tähän selaimeen (localStorage). Tarvitaan jotta "Vahvista viikko" voi committaa reseptit.json:n päivitykset.

Luominen: GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token. Scope: Only select repositories → perheen-ruokalista. Permissions: Contents → Read and write.

    Tietoa sivusta — käyttöohjeet ja automaatiot

    Mikä tämä sivu on

    Perheen viikkoruokalista on staattinen sivu, joka kokoaa yhteen kolmen viikon päivällisehdotukset, päiväkodin lounaslistan, ostoslistan ja Tampereen perhetapahtumat. Sivu pyörii GitHub Pagesin päällä ilman palvelinta — kaikki näkyvä data tulee joko HTML:ään upotettuna (auto-update generoi sen joka aamu) tai paikallisesti selaimen muistiin tallennetuista valinnoista.

    Välilehdet

    • Päivälliset: kahden seuraavan viikon koonti. Tästä näkymästä vahvistat kokkaukset (ks. alla).
    • Viikko 18 / 19 / 20: yksi viikko kerrallaan, kaikki ateriat (lounas vkl, päivällinen, välipala, iltapala). Klikkaa solua muokataksesi — muutokset tallentuvat selaimen muistiin automaattisesti. Klikkaa reseptin nimeä avataksesi reseptikortin (ainekset, ohjeet, palaute).
    • Ostoslista: viikon päivällisten ainekset koottuna, kategorioittain. "Päivitä reseptien mukaan" -nappi koostaa listan, säilyttäen käsin lisäämäsi rivit. Mobiilissa kassakäyntimoodi ja S-kaupat-hakulinkit per rivi.
    • Muistiinpanot: vapaa muistio, tallentuu selaimeen.
    • Tapahtumat: erillinen sivu Tampereen perhetapahtumille (Bubster API).

    Päivällisehdotukset ja kierrätys

    Algoritmi (julkaise.py) ehdottaa joka aamu kolmen viikon päivälliset reseptipoolista. Periaate: priorisoi reseptejä joita ei ole keitetty pitkään aikaan (last_cooked = null tai vanhin ensin), suosii arkireseptejä joiden valmistusaika on alle 35 min, välttää saman pääraaka-aineen toistumista peräkkäin. Kaksiportainen valinta: ensin tuoreet (yli 7 päivää last_cookedista), sitten täydennys vanhemmista jos pool ei riitä.

    Käyttäjän käsin asettamat valinnat (esim. kun klikkaat solua ja vaihdat reseptin) voittavat aina algoritmin ehdotuksen. Käyttäjäkohtainen tila tallentuu selaimen localStorageiin avaimella perheen-ruokalista-v3.

    Vahvista vk:n kokkaukset

    Kun viikon päivälliset on syöty, klikkaa ✓ Vahvista vk:n kokkaukset Päivälliset-välilehdellä. Sivu lukee localStoragen valinnat, hakee reseptit.json:n GitHubista, päivittää siihen last_cooked-arvot ja committaa muutokset takaisin. Tämän jälkeen seuraava auto-update välttää vahvistettuja reseptejä uudelle ehdotuspohjalle (7 päivän hard-cutoff).

    Toiminto vaatii GitHub Personal Access Tokenin. Asetus tehdään kerran sivun ⚙-napista:

    • GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token
    • Repository access: Only select repositories → perheen-ruokalista
    • Permissions: Contents → Read and write
    • Expiration: esim. 1 vuosi
    • Kopioi token (alkaa github_pat_) ja liitä sivun ⚙-napin avaamaan kenttään

    Token tallennetaan vain tähän selaimeen (localStorage, avain github_pat). Suosittelen säilyttämään tokenin myös salasanahallintaohjelmassa varmuuskopiona — jos selaimen data tyhjenee tai vaihdat laitetta, tarvitset sen uudelleen tai joudut luomaan uuden GitHubista.

    Automaatiot

    • Auto-update site (GitHub Actions, joka aamu klo 05:00 UTC = 08:00 Suomen aikaa): ajaa update_in_place.py, joka päivittää aikaleiman, päiväkodin valikon, Tampereen tapahtumat, kolmen viikon päivällisehdotukset (DEFAULT_CELLS) sekä commitaa muutokset päähaaraan.
    • Päiväkodin valikko: voimia.fi:n Aromi-API → Takahuhdin päiväkodin lounaat seuraavalle 14 päivälle. Upotetaan HTML:ään, näkyy päivän kohdalla "🏫 Päiväkoti · …" -rivinä.
    • Tapahtumat: tapahtumat.tampere.fi:n Bubster-API → tulevien 3 kuukauden lapsiperheille soveltuvat tapahtumat tapahtumat-sivulle.
    • Tilaushistoria (Windows Task Scheduler kotikoneella): seuraa S-kaupat-tilausten PDF-yhteenvetoja, parsii tuotteet ja hinnat, generoi historia.json:n joka näkyy ostoslistan hinta-arvioissa "aiemmin ostettu" -tooltipissä.

    Tekninen pohja

    Sivu on staattinen HTML, ei palvelinta. Kaikki dynaamisuus perustuu joko (a) HTML:ään upotettuun JSON-dataan, jonka auto-update kirjoittaa, tai (b) JavaScriptin localStorageiin, joka on selainkohtainen. Tämä tarkoittaa että jokaisella perheenjäsenellä on oma "tila" omassa selaimessaan — Heikin valinnat eivät synkronoidu suoraan Johannan puhelimelle eivätkä päinvastoin. Vahvista vk:n kokkaukset -toiminto on ainoa, joka kirjoittaa tilaa takaisin GitHubiin (ja sitä kautta kaikille).
